# Описание сценариев

## Сценарий 1. Успешный ответ ML

**Предусловия**

- Пользователь авторизован.
- `tripId` существует и доступен пользователю.
- `ML Recommendation Service` доступен.
- ML отвечает менее чем за 3 секунды.
- `confidence` не ниже настроенного порога.

**Основной поток**

1. Client Application отправляет `POST /api/v1/recommendations`.
2. `Recommendation Service` проверяет обязательные поля и права доступа.
3. `Recommendation Service` вызывает `ML Recommendation Service`.
4. ML возвращает список рекомендаций и достаточный `confidence`.
5. `Recommendation Service` приводит ответ к единому формату.
6. Клиент получает `200`, `source = ML`.

**Альтернативный поток**

- Если ML-ответ некорректен, пустой или `confidence` ниже порога, выполняется fallback-сценарий.

**Результат**

- Рекомендации сформированы на основе ML.

**Ответ пользователю**

- Список отелей, активностей или маршрутов без технических деталей работы ML.

## Сценарий 2. ML-сервис недоступен

**Предусловия**

- Запрос пользователя корректен.
- При вызове ML возникает сетевая ошибка или HTTP 5xx.

**Основной поток**

1. Client Application отправляет запрос.
2. `Recommendation Service` вызывает ML.
3. Возникает сетевая ошибка или HTTP 5xx.
4. `Recommendation Service` фиксирует причину `ML_NETWORK_ERROR` или `ML_HTTP_5XX`.
5. Запускается `Fallback Recommendation Module`.
6. Fallback получает профиль, историю и популярность.
7. Возвращается `200`, `source = FALLBACK`.

**Альтернативный поток**

- Если часть внутренних источников недоступна, применяется частичная деградация.
- Если все внутренние источники недоступны и рекомендации сформировать нельзя, возвращается `503`.

**Результат**

- Пользовательский сценарий продолжается без ожидания восстановления ML.

**Ответ пользователю**

- Рекомендации в обычном формате. Сообщение о поломке ML не показывается.

## Сценарий 3. Тайм-аут ML

**Предусловия**

- Запрос пользователя корректен.
- ML не отвечает в течение 3 секунд.

**Основной поток**

1. `Recommendation Service` отправляет запрос в ML.
2. Запускается таймер ожидания 3 секунды.
3. ML не возвращает ответ в срок.
4. `Recommendation Service` прекращает ожидание.
5. Фиксируется причина `ML_TIMEOUT`.
6. Запускается fallback-алгоритм.
7. Клиент получает `200`, `source = FALLBACK`.

**Альтернативный поток**

- Если fallback не смог сформировать рекомендации, возвращается `503`.

**Результат**

- Система не зависает на медленном ML.

**Ответ пользователю**

- Рекомендации, сформированные резервным алгоритмом.

## Сценарий 4. Низкий confidence

**Предусловия**

- ML доступен и вернул HTTP 200.
- Ответ ML синтаксически корректен.
- `confidence` ниже настроенного порога.

**Основной поток**

1. `Recommendation Service` получает ML-ответ.
2. Проверяет `confidence`.
3. Определяет, что `confidence` ниже порога.
4. Фиксирует причину `LOW_CONFIDENCE`.
5. Запускает fallback-алгоритм.
6. Клиент получает `200`, `source = FALLBACK`.

**Альтернативный поток**

- Если fallback вернул меньше объектов, чем запрошено в `limit`, клиент получает доступные объекты.

**Результат**

- Пользователь получает более надежный результат, чем низкоуверенный ML-ответ.

**Ответ пользователю**

- Рекомендации без технического упоминания низкого `confidence`.

## Сценарий 5. Недостаточно данных

**Предусловия**

- Запрос корректен.
- Для пользователя нет профиля или истории.
- `Recommendation Service` перешел на fallback или использует fallback по причине отказа ML.

**Основной поток**

1. Fallback пытается получить профиль.
2. Если профиль отсутствует, продолжает без него.
3. Fallback пытается получить историю.
4. Если история отсутствует, продолжает без нее.
5. Fallback получает популярность по направлению и типу рекомендации.
6. Формирует список по параметрам поездки и популярности.
7. Возвращается `200`, `source = FALLBACK`.

**Альтернативный поток**

- Если недоступен и `Popularity Service`, и нет данных профиля/истории, возвращается `503`.

**Результат**

- Новый пользователь все равно получает базовые рекомендации.

**Ответ пользователю**

- Популярные и подходящие по поездке объекты.
