# API-контракт

## Назначение

API формирует рекомендации для пользователя по поездке. Основной источник - `ML Recommendation Service`. При недоступности ML, тайм-ауте, низком `confidence` или некорректном ML-ответе возвращаются fallback-рекомендации в том же формате.

## Метод и URL

```http
POST /api/v1/recommendations
```

## Заголовки

| Заголовок | Обязателен | Описание |
|---|---:|---|
| `Content-Type: application/json` | Да | Формат тела запроса. |
| `Authorization: Bearer <token>` | Да | Токен пользователя. |
| `X-Trace-Id: uuid` | Нет | Технический идентификатор трассировки. Если не передан, создается сервером. |

## Поля запроса

| Поле | Тип | Обязательное | Описание |
|---|---|---:|---|
| `requestId` | string, uuid | Да | Идентификатор запроса на стороне клиента. |
| `userId` | string, uuid | Да | Идентификатор пользователя. Должен соответствовать авторизованному пользователю или правам клиента. |
| `tripId` | string, uuid | Да | Идентификатор поездки. |
| `recommendationType` | string | Да | Тип рекомендаций: `HOTEL`, `ACTIVITY`, `ROUTE`. |
| `limit` | integer | Нет | Количество рекомендаций: от 1 до 50, по умолчанию 10. |

## Пример запроса

```json
{
  "requestId": "2d6ef5fb-19c6-4c6a-8e2e-8b0d3cc30111",
  "userId": "a9f0b7a8-6506-47a8-86f5-2bbd8e7b1111",
  "tripId": "a4e42eb7-2acb-460a-ae70-9184f4f31111",
  "recommendationType": "HOTEL",
  "limit": 10
}
```

## Поля ответа

| Поле | Тип | Описание |
|---|---|---|
| `requestId` | string, uuid | Идентификатор исходного запроса. |
| `items` | array | Список рекомендаций. |
| `items[].entityId` | string, uuid | Идентификатор рекомендуемого объекта. |
| `items[].entityType` | string | Тип объекта: `HOTEL`, `ACTIVITY`, `ROUTE`. |
| `items[].title` | string | Название объекта. |
| `items[].score` | number | Оценка релевантности от `0` до `1`. |
| `items[].reasonCodes` | array[string] | Причины рекомендации. |
| `source` | string | Источник результата: `ML` или `FALLBACK`. |
| `generatedAt` | string, date-time | Время формирования ответа. |

## Пример ответа

```json
{
  "requestId": "2d6ef5fb-19c6-4c6a-8e2e-8b0d3cc30111",
  "items": [
    {
      "entityId": "f6b6dc7e-6c11-4928-a370-118cb1111111",
      "entityType": "HOTEL",
      "title": "Hotel Example",
      "score": 0.87,
      "reasonCodes": [
        "MATCHES_USER_INTERESTS",
        "POPULAR_IN_DESTINATION"
      ]
    }
  ],
  "source": "FALLBACK",
  "generatedAt": "2026-06-29T12:00:00Z"
}
```

## HTTP-коды

| Код | Когда возвращается |
|---:|---|
| 200 | Рекомендации сформированы через ML или fallback. |
| 400 | Некорректный запрос. |
| 401 | Пользователь не авторизован. |
| 403 | Нет доступа к пользователю или поездке. |
| 404 | Пользователь или поездка не найдены. |
| 500 | Внутренняя ошибка `Recommendation Service`. |
| 503 | Рекомендации невозможно сформировать ни основным, ни резервным способом. |

## Формат ошибки

```json
{
  "code": "INVALID_REQUEST",
  "message": "Некорректные параметры запроса",
  "traceId": "5dc37a1b-5917-4571-971a-a8a3dd111111"
}
```

## Коды ошибок

| Код | Описание |
|---|---|
| `INVALID_REQUEST` | Ошибка в параметрах запроса. |
| `UNAUTHORIZED` | Пользователь не авторизован. |
| `FORBIDDEN` | Нет доступа к ресурсу. |
| `NOT_FOUND` | Пользователь или поездка не найдены. |
| `INTERNAL_ERROR` | Внутренняя ошибка сервиса. |
| `RECOMMENDATIONS_UNAVAILABLE` | Рекомендации невозможно сформировать. |

Технические детали отказа ML пользователю не возвращаются.
