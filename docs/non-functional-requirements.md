# Нефункциональные требования

| ID | Требование | Проверка |
|---|---|---|
| NFR-01 | Тайм-аут ожидания `ML Recommendation Service` - 3 секунды. | Тест с ML-ответом позже 3 секунд приводит к fallback. |
| NFR-02 | Общий ответ системы не должен зависать при отказе ML. | При сетевой ошибке ML API возвращает ответ или контролируемую ошибку. |
| NFR-03 | Резервный алгоритм должен иметь отдельный временной бюджет: 1.5 секунды. | При превышении бюджета fallback завершается контролируемой ошибкой. |
| NFR-04 | Система должна логировать причины переключения на fallback. | В логах есть `fallbackReason`. |
| NFR-05 | API должен сохранять обратную совместимость. | Поля ответа не меняются при источниках `ML` и `FALLBACK`. |
| NFR-06 | Паспортные и платежные данные не должны передаваться в рекомендательные сервисы. | В контрактах и логах отсутствуют такие поля. |
| NFR-07 | Технические детали отказа ML не должны показываться пользователю. | Ответы API не содержат причин `ML_TIMEOUT`, `ML_HTTP_5XX` и подобных кодов. |
| NFR-08 | Целевое общее время ответа API - до 5 секунд при доступных внутренних источниках. | Нагрузочный или интеграционный тест. |
| NFR-09 | Доступность API рекомендаций - не ниже 99.5% в месяц при доступности внутренних источников. | Мониторинг uptime. |

## Логирование

Минимальный набор полей:

- `requestId`;
- `traceId`;
- `recommendationType`;
- `source`;
- `fallbackReason`;
- время ответа ML;
- общее время ответа;
- количество рекомендаций;
- код ошибки.

Не логировать:

- паспортные данные;
- платежные данные;
- полный профиль пользователя;
- персональные данные в открытом виде;
- готовые ML-предсказания для повторного использования.

## Метрики

| Метрика | Назначение |
|---|---|
| `recommendation.requests.count` | Количество запросов. |
| `recommendation.response.time.avg` | Среднее время ответа. |
| `recommendation.ml.timeouts.count` | Количество тайм-аутов ML. |
| `recommendation.ml.errors.count` | Количество ошибок ML. |
| `recommendation.fallback.rate` | Доля fallback-ответов. |
| `recommendation.low_confidence.count` | Количество низких `confidence`. |
| `recommendation.fallback.errors.count` | Количество ошибок fallback. |
| `recommendation.empty_results.count` | Количество пустых результатов. |
